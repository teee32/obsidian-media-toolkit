# Issue 9: 删除交互不专业 - 使用浏览器 confirm()

## 严重程度: P2 - UX 问题

### 问题描述
UnreferencedImagesView 的删除交互存在问题：

1. **单张删除**: 使用 `confirm()`
2. **批量删除**: 也使用 `confirm()`，同步循环删除

### 影响
- 在 Obsidian 生态里，这种删除 UX 一般会被认为"不够专业 / 不够安全"
- 批量删除没有进度提示
- 错误只统计数量，没有逐项可追溯的恢复路径

### 修复建议
1. **使用 Obsidian 的 Modals**:
   - 创建确认 Modal，显示即将删除的文件列表
   - 提供"全选"、"取消全选"等交互

2. **异步删除 + 进度条**:
   - 使用 async/await
   - 显示删除进度
   - 记录每个删除操作的结果

3. **错误处理**:
   - 详细记录每个失败的文件和原因
   - 提供"重试失败项"功能
